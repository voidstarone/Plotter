NoteTaker Library API Documentation

Overview

The NoteTaker library provides a hierarchical note-taking system with three main entities: Projects, Folders, and Notes. The library supports unlimited nesting of folders, arbitrary metadata attributes on notes, and comprehensive search and management capabilities.

Architecture

Project (Root Container)
├── Folder (Can contain subfolders and notes)
│ ├── Subfolder (Unlimited nesting depth)
│ │ └── Note (With arbitrary attributes)
│ └── Note (With arbitrary attributes)
└── Folder
└── Note

Core Classes

Project Class

The Project class serves as the root container for organizing folders and notes.

Key Features

  • Automatic unique ID generation
  • Folder management with search capabilities
  • Tree visualization support
  • Memory management through smart pointers

Constructor Examples

// Create default project
auto project = std::make_shared<Project>();
// Create named project
auto project = std::make_shared<Project>("My Research");
// Create project with description
auto project = std::make_shared<Project>("My Research", "Academic research notes");

Basic Operations

// Add folders
auto folder1 = project->addFolder("Literature Review");
auto folder2 = project->addFolder("Ideas", "Creative thoughts and concepts");
// Find folders
auto found = project->findFolderByName("Literature Review");
auto foundById = project->findFolder("folder_12345");
// Get project information
std::cout << "Project: " << project->getName() << std::endl;
std::cout << "Folders: " << project->getFolderCount() << std::endl;
// Display tree structure
std::cout << project->toTreeString() << std::endl;

Folder Class

The Folder class provides hierarchical organization with support for both notes and subfolders.

Key Features

  • Support for nested folder structures (unlimited depth)
  • Note management with search capabilities
  • Subfolder management with search capabilities
  • Recursive operations (e.g., total note counting)
  • Tree visualization support

Constructor Examples

// Create default folder
auto folder = std::make_shared<Folder>();
// Create named folder
auto folder = std::make_shared<Folder>("Research Papers");
// Create folder with description
auto folder = std::make_shared<Folder>("Research Papers", "Academic literature collection");

Note Management

// Add notes
auto note1 = folder->addNote("Paper Title");
auto note2 = folder->addNote("Another Paper", "Initial content here");
// Find notes
auto found = folder->findNoteByTitle("Paper Title");
auto foundById = folder->findNote("note_12345");
// Remove notes
bool removed = folder->removeNote("note_12345");
// Get note counts
size_t directNotes = folder->getNoteCount();
size_t totalNotes = folder->getTotalNoteCount(); // Includes subfolders

Subfolder Management

// Create nested structure
auto csFolder = folder->addSubfolder("Computer Science");
auto aiFolder = csFolder->addSubfolder("Artificial Intelligence", "AI research papers");
// Find subfolders
auto found = folder->findSubfolderByName("Computer Science");
auto foundById = folder->findSubfolder("folder_67890");
// Get subfolder count
size_t subfolderCount = folder->getSubfolderCount();

Note Class

The Note class provides flexible content storage with arbitrary metadata attributes.

Key Features

  • Automatic unique ID generation
  • Automatic creation and modification timestamps
  • Type-safe arbitrary attribute system using std::any
  • Template-based attribute access with type checking
  • Support for any C++ type as attribute values

Constructor Examples

// Create default note
auto note = std::make_shared<Note>();
// Create titled note
auto note = std::make_shared<Note>("Research Ideas");
// Create note with content
auto note = std::make_shared<Note>("Paper Summary", "This paper discusses...");

Content Management

// Update content
note->setTitle("Updated Title");
note->setContent("New content here");
// Get content
std::string title = note->getTitle();
std::string content = note->getContent();
std::string id = note->getId();
// Get timestamps
auto created = note->getCreatedAt();
auto updated = note->getUpdatedAt();
std::string createdStr = note->getCreatedAtString();
std::string updatedStr = note->getUpdatedAtString();

Attribute System

The attribute system allows storing arbitrary key-value pairs with any C++ type:

// Set various attribute types
note->setAttribute<std::string>("author", "John Doe");
note->setAttribute<int>("priority", 5);
note->setAttribute<double>("rating", 4.7);
note->setAttribute<bool>("completed", false);
note->setAttribute<std::vector<std::string>>("tags", {"research", "ai", "important"});
// Get attributes (throws exception if not found or wrong type)
std::string author = note->getAttribute<std::string>("author");
int priority = note->getAttribute<int>("priority");
// Get attributes with default values (safe, no exceptions)
std::string author = note->getAttribute<std::string>("author", "Unknown");
int priority = note->getAttribute<int>("priority", 0);
bool urgent = note->getAttribute<bool>("urgent", false);
// Check and manage attributes
bool hasAuthor = note->hasAttribute("author");
bool removed = note->removeAttribute("old_attribute");
std::vector<std::string> keys = note->getAttributeKeys();
size_t count = note->getAttributeCount();
// Clear all attributes
note->clearAttributes();

Memory Management

The library uses smart pointers for automatic memory management:

  • std::shared_ptr for all entities
  • Automatic cleanup when objects go out of scope
  • No manual memory management required
  • Safe sharing of objects between containers

Error Handling

Exceptions

  • std::runtime_error: Thrown when attribute access fails
  • std::bad_any_cast: Thrown when attribute type conversion fails

Safe Methods

  • Use attribute methods with default values to avoid exceptions
  • Check return values from find methods (return nullptr if not found)
  • Use hasAttribute() before accessing attributes

Best Practices

1. Memory Management

// Good: Use smart pointers
auto project = std::make_shared<Project>("My Project");
auto folder = project->addFolder("Documents");
// The library handles cleanup automatically

2. Attribute Access

// Safe attribute access with defaults
std::string author = note->getAttribute<std::string>("author", "Unknown");
int priority = note->getAttribute<int>("priority", 0);
// Check before accessing
if (note->hasAttribute("important")) {
bool isImportant = note->getAttribute<bool>("important");
// Process...
}

3. Search Operations

// Always check return values from find operations
auto folder = project->findFolderByName("Research");
if (folder) {
auto note = folder->findNoteByTitle("Important Paper");
if (note) {
// Use note...
}
}

4. Hierarchical Navigation

// Navigate through nested structures safely
auto lit = project->findFolderByName("Literature");
if (lit) {
auto cs = lit->findSubfolderByName("Computer Science");
if (cs) {
auto ai = cs->findSubfolderByName("AI");
if (ai) {
// Access AI folder contents
}
}
}

Threading Considerations

The library is not thread-safe. If used in a multi-threaded environment:

  • Protect access with mutexes
  • Consider read-write locks for performance
  • Be careful with shared_ptr reference counting

Performance Notes

  • Searching is O(n) for linear searches by name
  • Tree traversal is O(n) where n is total number of nodes
  • Attribute access is O(1) average case (hash map)
  • Memory usage scales linearly with content
